# 核心功能清单

## 需求洞察

目标是将每天 B 站 GitHub 热榜视频转化为可阅读、可积累、与个人学习方向相关的本地项目分析，而不是仅收集链接。

## 场景走查

用户运行 `python orchestrator.py` 后，系统获取最新视频中的项目，跳过已分析的同类项目，完成分析和排序；用户可逐项查看概览或完整报告。网络、克隆、模型调用或项目内容缺失时，必须给出可定位的错误并继续处理其余项目。

## 功能

- 热榜发现与克隆：从指定 UP 主最新视频简介提取去重后的前十个 GitHub 仓库，并在本地克隆或跳过已有仓库。
- 项目代码扫描：读取 README、目录和关键源码，生成可供各分析器共享的结构化项目资料。
- 四路并行分析：对每个项目并行生成概览、架构骨架、Agent 工程评估和裂隙分析。
- 串行综合分析：基于前序结果生成唯物辩证法分析和个人适用性分析。
- 分类与索引积累：追加六维度分类发现，维护项目索引并识别同类项目差异。
- 相关性排序与交互展示：按个人画像排序，在命令行逐项展示并支持展开、跳过和退出。
- 本地报告沉淀：将完整五视角分析写为 Markdown 报告。
